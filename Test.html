【概論】問題 01：オブジェクト指向の特徴と基本的概念を知っている。
問 01
オブジェクト指向の概念を取り入れたアプリ開発について、最適な説明をア〜ウの中から答えなさい。
  • ア大規模なプログラムになると人間がプログラムの内容を把握しきれなくなるため、現実世界を真似してプログラムを部品化する。
  • イ大規模なプログラムになるとコンピュータの能力が限界を迎えるため、現実世界を真似してプログラムを部品化する。
  • ウ大規模なプログラムになると開発期間が長くなってしまうため、コンピュータの能力に合わせてプログラムを部品化することで開発期間の短縮化を図る。
💡 解説
✅ 正解：ア
オブジェクト指向は「人間の認識・把握能力の限界」を補うために生まれた概念です。大規模なプログラムになると、人間が全体を把握できなくなるため、現実世界の「もの（オブジェクト）」に対応した形でプログラムを部品化します。コンピュータの性能とは無関係であり、開発期間短縮が主目的でもありません。
問 02
オブジェクト指向の概念を取り入れたプログラミングの手法について、最適な説明をア〜ウの中から答えなさい。
  • ア実行手順に従ってコードを記述する方法が「オブジェクト指向」プログラミングで、プログラムを部品化した各オブジェクトに責務を持たせる方法が「手続き型」プログラミングである。
  • イ実行手順に従ってコードを記述する方法が「手続き型」プログラミングで、プログラムを部品化した各オブジェクトに実行手順を記述する方法が「オブジェクト指向」プログラミングである。
  • ウ実行手順に従ってコードを記述する方法が「手続き型」プログラミングで、プログラムを部品化した各オブジェクトに責務を持たせる方法が「オブジェクト指向」プログラミングである。
💡 解説
✅ 正解：ウ
「手続き型」は実行順序に沿ってコードを記述するスタイルです。「オブジェクト指向」は現実世界のものに対応したオブジェクトを部品化し、それぞれのオブジェクトに責務を持たせるスタイルです。イは「責務」ではなく「実行手順」としている点が誤りです。
問 03
オブジェクトの構成要素について、最適な説明をア〜ウの中から答えなさい。
  • ア構成要素は、他のオブジェクトを所有する「集約」と関係性を表す「汎化」の2つがある。
  • イ構成要素は、情報保持責任を持つ「属性」と行動責任を持つ「操作」の2つがある。
  • ウ構成要素は、開発者の裁量に任され、自由にいくつでも用意することができる。
💡 解説
✅ 正解：イ
オブジェクトの構成要素は「属性（フィールド）」と「操作（メソッド）」の2つです。属性は情報（データ）を保持する責任を持ち、操作は行動（処理）の責任を持ちます。「集約」「汎化」はオブジェクト間の関係を表すUMLの概念であり、オブジェクト自体の構成要素ではありません。
問 04
オブジェクトの相互作用について、最適な説明をア〜ウの中から答えなさい。
  • アオブジェクトから他のオブジェクトの操作を呼び出して相互に作用してプログラムとして動く。他のオブジェクトの情報はカプセル化により守られているので、相互作用することはない。
  • イオブジェクトは完全に独立しているため、オブジェクト同士の相互作用を必要とせず単独で動く。
  • ウオブジェクトから他のオブジェクトの操作の呼び出し、他のオブジェクトの情報の取得や変更を行うことによってオブジェクトは相互に作用し、全体で1つのプログラムとして動く。
💡 解説
✅ 正解：ウ
オブジェクト指向では、複数のオブジェクトがメッセージ（メソッド呼び出し）を通じて相互作用し、全体として1つのプログラムとして機能します。カプセル化は情報を外部から守る仕組みですが、getter/setter などを通じて情報の取得・変更は行えます。
問 05
オブジェクト指向言語の3大機能を記述しなさい（完答）。
💡 解説
✅ 正解：カプセル化・継承・ポリモーフィズム（多態性）

問 09
デフォルトコンストラクタについて、最適な説明をア〜ウの中から答えなさい。
  • アコンストラクタが未定義の場合に、引数なし、処理なしのコンストラクタがコンパイル時に自動生成される。
  • イコンストラクタが未定義の場合に、フィールドの数に合わせて自動的に引数あり、処理ありのコンストラクタがコンパイル時に自動生成される。
  • ウクラスにフィールドが未定義の場合に、引数なし、処理なしのコンストラクタがコンパイル時に自動生成される。
💡 解説
✅ 正解：ア
デフォルトコンストラクタは「コンストラクタが1つも定義されていない場合」に自動生成されます。内容は引数なし・処理なしです。フィールドの有無は関係ありません。なお、コンストラクタを1つでも定義すると、デフォルトコンストラクタは自動生成されなくなります。

問題 02：カプセル化概念を理解し、プログラミングを行うことができる。
問 11
カプセル化について、最適な説明をア〜オの中から答えなさい。
  • ア外部のクラスに公開するメンバーと非公開とするメンバーを区別せず、すべてのメンバーを外部からアクセスできないようにする。
  • イ外部のクラスに公開するメンバーと非公開とするメンバーを分けて制限し、非公開メンバーのみ外部からアクセスできるようにする。
  • ウ外部のクラスに公開するメンバーと非公開とするメンバーを分けて制限し、非公開メンバーに public 修飾子をつけて外部からアクセスできるようにする。
  • エ外部のクラスに公開するメンバーと非公開とするメンバーを分けて制限し、公開されているメンバーのみ外部からアクセスできるようにする。
  • オ外部のクラスに公開するメンバーと非公開とするメンバーを分けて制限するが、すべてのメンバーに外部から直接アクセスできるようにする。
💡 解説
✅ 正解：エ
カプセル化とは、公開するメンバー（public）と非公開にするメンバー（private）を分けて制限し、公開されているメンバーのみ外部からアクセスできるようにする仕組みです。
問 12
カプセル化のメリットについて、不適切な説明をア〜オの中から答えなさい。
  • アクラス内の情報を利用者からの不用意なアクセスから守ることで、インスタンスの独立性を保つことができる。
  • イクラス内のフィールドに対して、読み取り専用、書き込み専用のフィールドを利用者に対して実現できる。
  • ウクラスの内部設計（たとえば、フィールドの名前など）を自由に変更できる。
  • エ利用者が指定した、フィールドに設定する値の妥当性を検査できる。
  • オ利用者がクラスの内部設計を理解していなくてもクラスを使用できる。
💡 解説
✅ 正解（不適切な説明）：イ
イが不適切です。カプセル化では「フィールド自体」を読み取り専用・書き込み専用にするのではなく、getter のみ提供（読み取り専用）または setter のみ提供（書き込み専用）という形でアクセスを制限します。ア・ウ・エ・オはすべてカプセル化の正しいメリットです。

問題 03：継承を理解し、プログラミングを行うことができる。
問 16
Java のクラスの継承について、最適な説明をア〜オの中から答えなさい。
  • ア単一継承と多重継承の2つがある。
  • イ多重継承のみで単一継承は認めていない。
  • ウ単一継承のみで多重継承は認めていない。
  • エ多重継承を行っても良いが、単一継承が推奨されている。

✅ 正解：ウ

  • イfinal が付けられたメソッドは、オーバーライドできない。

問 20
下記の Hero クラスを継承して SuperHero クラスを宣言し、attack メソッドをオーバーライドする時、コンパイルエラーにならない記述をア〜オの中から答えなさい。

問題 04：継承・抽象クラスを使用することができる。
問 21
抽象クラスについて、不適切な記述をア〜オの中から答えなさい。
  • ア抽象クラス型の変数を宣言することができる。
  • イフィールドを持つことができる。
  • ウインスタンスを生成することができない。
  • エ抽象メソッドだけが宣言できる。

💡 解説
✅ 正解（不適切な記述）：エ
エが不適切です。抽象クラスは抽象メソッドだけでなく、通常のメソッド（具体的な実装を持つメソッド）も持つことができます。フィールド・通常メソッド・抽象メソッドを混在できます。
問 22
protected 修飾子について、最適な記述をア〜オの中から答えなさい。
  • アメンバーへのアクセスを許可する範囲は、すべてのクラスである。
  • イメンバーへのアクセスを許可する範囲は、自分と同じパッケージに属するか、自分を継承した子クラスである。
  • ウメンバーへのアクセス制御だけではなく、クラスへのアクセス制御にも指定できる。
  • エメンバーへのアクセスを許可する範囲は、自分自身のクラスのみである。
  • オメンバーへのアクセスを許可する範囲は、自分と同じパッケージに属するクラスである。
💡 解説
✅ 正解：イ
アクセス修飾子の範囲（広い順）：
public：すべてのクラス。
protected：同じパッケージ ＋ 継承した子クラス。
なし（デフォルト）：同じパッケージのみ。
private：自分自身のクラスのみ。
問 23
抽象クラスと継承について、最適な記述をア〜オの中から答えなさい。
  • ア抽象クラスを継承したクラスは、必ず抽象メソッドをオーバーライドしなくてはならない。
  • イ抽象クラスを継承したクラスは、final を宣言すれば抽象メソッドをオーバーライドしなくて良い。
  • ウ抽象クラスを継承したクラスは、abstract を宣言すれば抽象メソッドをオーバーライドしなくて良い。
  • エ抽象クラスを継承したクラスは、protected を宣言すれば抽象メソッドをオーバーライドしなくて良い。
  • オ抽象クラスを継承したクラスは、抽象メソッドを無視して良い。
💡 解説
✅ 正解：ウ
抽象クラスを継承した場合、通常はすべての抽象メソッドをオーバーライドしなければなりません。ただし、子クラス自身も abstract クラスとして宣言すれば、抽象メソッドのオーバーライドを孫クラス以降に委ねることができます。
問 24
下記の Hero クラスの run メソッドを抽象メソッドに変更する場合、最適なコードをア〜オの中から答えなさい。
public abstract class Hero { public void run() { System.out.println("勇者は逃げ出した。") ; } }
  • アpublic void run() ;
  • イprotected void run() ;
  • ウpublic abstract void run() { System.out.println("勇者は逃げ出した。") ; }
  • エpublic abstract void run ;
  • オpublic void abstract run ;
💡 解説
✅ 正解：エ
抽象メソッドは「abstract キーワードを付け、メソッド本体（{ }）を持たずセミコロンで終わる」形式です。
ア：abstract がない。
イ：アクセス修飾子が変わり、abstract もない。
ウ：abstract メソッドにメソッド本体は書けない（コンパイルエラー）。
エ：正しい抽象メソッドの宣言。
オ：abstract の位置が誤り（戻り値型の前に書く）。
問 25
下記の Character クラスと Hero クラスの継承について、最適なコードをア〜ウの中から答えなさい。
  • ア
    public class Character { public abstract void run() ; } public class Hero extends Character { public void run() { System.out.println("勇者は逃げ出した。") ; } }
  • イ
    public abstract class Character { public abstract void run() ; } public class Hero extends Character { public void run() { System.out.println("勇者は逃げ出した。") ; } }
  • ウ
    public abstract class Character { public void run() ; } public class Hero extends Character { public void run() { System.out.println("勇者は逃げ出した。") ; } }
💡 解説
✅ 正解：イ
ア：抽象メソッドを含むクラスは必ず abstract クラスにしなければなりません。class Character に abstract がないためコンパイルエラー。
イ：abstract class Character に abstract void run() を宣言し、Hero クラスが正しくオーバーライドしている → 正解。
ウ：public void run() ; は abstract キーワードがなく本体もないためコンパイルエラー。
